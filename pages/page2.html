Build a basic HTML document in vision and open a new repo related to it in github.

Creating a repo/ adding the repo on git hub and cloning it.
Go to your Github page-click on New Repository-Name it- thick the button: Add a read me file-click Create Repository.
Click on your Repository-click on Code-click on SSH-copy the SSH key.
Go to your Documents folder on your computer or the directory where is saved your GitHub folder-open Git folder-click right- click on open Gitbash here-type in the Gitbash Terminal git clone and paste the ssh key that you copied-enter.
Go to Visual Studio Code-click on File-click on Open Folder and start coding.

Put a image.

To put the image into your working project on Vs Code there is 2 ways:
either you drag it there
or go into the folder where you have saved the image-copy and paste on the files menu.
Don't forget always rename you image files,follow good practices and use CamelCase and visualise all the steps/ changes into your live server-click right-open with Live server- if your server don't work in real time just press refresh button to see the changes.

Link it's assets.

Click right on index.html-copy and paste into your menu bar.Now this file it will appear as index.html copy-click right and rename it page2.html.
Go to the project code from page2 in the body section at your hyperlink.
Your hyperlink you need to update it.
But before that lets build the structure to make sure that you can link it.Click on the icon from menu- new folder-name it pages or however you want-click right on the pages folder and open folder- name it assests, drag the picture into the assets folder.Open another folder-name it pages-move page2 into this folder-either right click choose option to move it or just drag it there.
Now the structure is there.Go back to index.html and update the link.
Oops.We have a error on live server "Error cannot get /pages/.index.html" Well why is that so.Firstly is looking for a sibling or subchild,have a research on Relationship between files.And secondly page2 is still set like index.html.So we need to go up on the folder directory.Like this: ../index.html.Remember, when you just want the current folder one dot is just enaugh.
Code is ready, we escape of the error, have a double check all the time in the HTML validator anyway.Good practice.
Go to readme.md file and write a description and save it.Most of the files have a U on the right side witch means untracked but now your readme file have a M which means modified.

Git add, commit and git push

You've done with everything in Vs Code now let's save the changes on Github page.Open the GitBash terminal (Ctrl+J)- type git add .- check if you are in the current folder- type git commit -am "demo". am - stands for a = all files and m = message.You can put any message.
